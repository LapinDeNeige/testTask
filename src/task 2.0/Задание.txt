Нормализация таблицы заключается в создании трёх дополнительных таблиц:
group_order для групповых заказов ,
preferential_order для льготных заказов и 
таблица other_orders - для остальных типов заказов

Эти таблицы имеют одинаковую структуру:
 event_id              | int(11)      | 
| ticket_adult_price    | int(11)      |
| ticket_adult_quantity | int(11)      |
| ticket_kid_price      | int(11)      | 
| ticket_kid_quantity   | int(11)      | 
| barcode               | varchar(120) | 
| equal_price           | int(11)      | 
| CREATED               | datetime     | 
| event_date            | datetime     | 
,однако таблица other_orders еще имеет колонку order_type типа VARCHAR(125) для указания типа билета

Данные таблицы были приведены к первой нормальной форме, в итоге получилось избежать избыточности данных путем 
создания отдельных таблиц и идентификации набора данных при помощи первичного ключа
